# Test Suites

A collection of organized test cases designed to validate specific functionality or features of software applications and APIs. Test suites group related test cases into logical units that can be executed together, providing comprehensive coverage of a system's behavior. They are widely used by developers to build, maintain, and scale software testing across functional testing, regression testing, contract testing, and compliance validation.

## APIs

| Name | Description |
|---|---|
| [Postman Collections API](https://www.postman.com/) | Primary organizational unit for grouping API test cases into test suites with programmatic management. |
| [JUnit 5](https://junit.org/junit5/) | Java testing framework for organizing test cases into suites with grouping, tagging, and parallel execution. |
| [pytest](https://pytest.org/) | Python testing framework supporting suite organization through classes, directories, markers, and fixtures. |
| [Jasmine](https://jasmine.github.io/) | BDD JavaScript framework organizing test cases into describe/it suite blocks for Node.js and browsers. |
| [Mocha](https://mochajs.org/) | Flexible JavaScript test suite framework supporting async tests with rich reporting and assertion library integration. |
| [Jest](https://jestjs.io/) | Zero-configuration JavaScript testing framework with built-in suite runner, mocking, and coverage reporting. |
| [Bruno](https://www.usebruno.com/) | Open-source API client storing test suites as plain files for git-based collaboration and CI/CD integration. |
| [Hurl](https://hurl.dev/) | Command-line tool running HTTP requests as lightweight plain-text API test suites in CI/CD pipelines. |
| [TestNG](https://testng.org/) | Java testing framework with advanced suite configuration including grouping, prioritization, and parallel execution. |
